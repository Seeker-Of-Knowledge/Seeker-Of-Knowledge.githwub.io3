----Cheating and modifying units

This document covers how you can modify your units, for example, to test a quest or an interaction you just written.

The main ingredient you need is the Javascript Console of your browser. Usually, you can open it via Ctrl + Shift + J. Once you're there, there should be an object called `SugarCube` --- if the object is there (e.g., by typing `SugarCube` in the console), you are in the right place.

### Generating quests / opportunities

Go to `Settings`, then turn on Debug Mode if you haven't. If you reload `Settings`, there should be a `Test Quest` link. Click there, find the quest you want to test, then click on the `(make instance)` to make an instance of that quest in the current game.

Warning: it may fail if it is not create-able, e.g., if you're creating a rescue quest when there are nobody to rescue.

### Money

Go to `Settings`, then turn on Debug Mode if you haven't. If you reload `Settings`, there should be a `Debug get money` link.

### Get Units IDs

Type `SugarCube.State.variables.unit`. It should then display all the units in the game. Find the one you need, e.g., the unit with id 2. Then you get it by assigning it to a variable, e.g.,

``` SugarCube.State.variables.unit[2] ```

Player character is special:

``` SugarCube.State.variables.unit.player ```
